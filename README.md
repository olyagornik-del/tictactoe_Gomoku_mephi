# Гомоку с переключаемыми ИИ-алгоритмами

Гомоку (5 в ряд) на условно-бесконечной доске. Человек играет против ИИ
и **в любой момент партии** может на лету переключить алгоритм противника
между четырьмя вариантами, наблюдая за их производительностью в боковой
панели. Есть режим «ИИ против ИИ» для бенчмарков.

- **Доска** бесконечная: `dict[(x, y) -> 'X' | 'O']`, растёт по мере ходов,
  координаты любые целые (в т.ч. отрицательные).
- **Победа** — 5 одинаковых символов подряд по горизонтали, вертикали или
  диагонали.
- **GUI** на pygame: камера со скроллом/зумом, переключатель алгоритма,
  ползунки настроек, живая таблица метрик.

---

## Установка

Нужен Python 3.11+ (разрабатывалось на 3.13).

```bash
python -m venv venv
source venv/bin/activate            # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Зависимости: `pygame`, `numpy`, `pytest`, `scikit-learn`
(последний — только для `train_test_split` при обучении перцептрона).

---

## Запуск игры

```bash
python main.py
```

По умолчанию: вы играете **крестиками (X)** и ходите первым, ИИ отвечает
**ноликами (O)**.

### Управление

| Действие | Как |
|---|---|
| Сделать ход | Левый клик по клетке |
| Панорамирование | Зажать **правую** кнопку мыши и тащить |
| Зум | Колесо мыши |
| Сменить алгоритм ИИ | Клик по радио-кнопке справа (в любой момент партии) |
| Настроить алгоритм | Ползунок «Глубина» (Minimax/AB) или «Симуляций» (MCTS) |
| Новая партия | Кнопка «Новая игра» (статистика **сохраняется**) |
| Авто-режим | Кнопка «ИИ vs ИИ» |
| Обнулить метрики | Кнопка «Сбросить статистику» |
| Выход | `Esc` или закрыть окно |

Переключение алгоритма **не сбрасывает** позицию — можно поменять
противника посреди партии и сравнить, как сыграют разные алгоритмы из
одной и той же позиции.

## Алгоритмы

Все четыре — отдельные классы, наследующие абстрактный `Agent`
(`agents/base.py`) с методом `choose_move(board) -> (x, y)`. Каждый агент
рассматривает только пустые клетки в радиусе 2 от уже сыгранных
(«окно поиска») — это держит ветвление обозримым.

### 1. Minimax (`agents/minimax.py`)
Полный перебор дерева ходов до фиксированной глубины (ползунок 1–6).
На листьях применяется эвристическая оценка позиции. Победа оценивается
с поправкой на остаточную глубину, поэтому более быстрые победы (и более
медленные поражения) ценятся выше. Метрика — число просмотренных узлов.

### 2. Alpha-Beta (`agents/alphabeta.py`)
Тот же Minimax с альфа-бета отсечением: ветви, не влияющие на результат,
отбрасываются. Ходы упорядочиваются от центра наружу — это усиливает
отсечения. При той же глубине играет так же, но просматривает кратно
меньше узлов.

### 3. MCTS (`agents/mcts.py`)
Monte Carlo Tree Search с UCB1 и настраиваемым числом симуляций
(ползунок 100–5000). Под бесконечную доску адаптирован: ветвление
ограничено локальным окном, доигровки (playout) управляемые
(берут немедленный выигрыш / блокируют угрозу / тянут самую длинную
линию), с лимитом глубины. Метрика — число симуляций.

### 4. Перцептрон (`agents/perceptron.py`)
Однослойный перцептрон на NumPy (`sigmoid` + бинарная кросс-энтропия).
Для каждого хода-кандидата формирует признаки — окно 9×9 вокруг хода,
one-hot по 3 каналам (пусто / свой / чужой) = 243 признака — прогоняет
сеть и выбирает ход с максимальным выходом. Метрика — число прогонов сети.
Веса грузятся из `models/perceptron.npz`; если файла нет, радио-кнопка
«Перцептрон» в GUI отключается (обучите модель — см. ниже).

### Эвристика (`agents/heuristic.py`)
Общая для Minimax и Alpha-Beta. Оценка позиции = сумма весов своих
паттернов минус веса паттернов оппонента по 4 направлениям. Веса вынесены
в константы и легко тюнятся:

| Паттерн | Вес |
|---|---|
| Пять подряд (победа) | 1 000 000 |
| Открытая четвёрка | 100 000 |
| Закрытая четвёрка | 10 000 |
| Открытая тройка | 5 000 |
| Закрытая тройка | 500 |
| Открытая двойка | 100 |
| Закрытая двойка | 10 |

## Боковая панель: метрики

После каждого хода ИИ обновляется таблица (4 строки — по одной на
алгоритм): **среднее время** хода, **последнее время** (мс) и **число
ходов** за сессию. Метрики накапливаются по всем партиям, пока не нажата
«Сбросить статистику». Профилирование — `metrics/profiler.py`,
агрегация — `metrics/stats.py`.

## Обучение перцептрона

Перцептрон учится на партиях Alpha-Beta vs Alpha-Beta (с 10% случайных
ходов для разнообразия). Метка хода = 1, если эвристическая оценка
позиции после хода положительна для сходившего, иначе 0.

```bash
# дефолты ТЗ: 2000 партий, 50 эпох, lr=0.1, batch=32 → models/perceptron.npz
python -m training.train_perceptron

# быстрый прогон с прогрессом
python -u -m training.train_perceptron --games 400 --epochs 100
```

Полезные флаги:

| Флаг | Назначение | Дефолт |
|---|---|---|
| `--games N` | число партий для генерации данных | 2000 |
| `--epochs N` | число эпох SGD | 50 |
| `--lr X` | learning rate | 0.1 |
| `--batch-size N` | размер мини-батча | 32 |
| `--init {zeros,small_random,large_random}` | инициализация весов | small_random |
| `--out PATH` | куда сохранить веса | models/perceptron.npz |
| `--cache-data PATH` | кэш датасета: есть файл — загрузит (генерация пропускается), нет — создаст | — |
| `--eval-games N` | партий против Random для быстрой проверки | 4 |

Каждый запуск дописывает строку в `training_log.md` (метрики train/val,
winrate против Random). Чтобы быстро перебирать **только эпохи** на
фиксированных данных, используйте кэш:

```bash
python -u -m training.train_perceptron --games 400 --epochs 50  --cache-data data/g400.npz --out models/e50.npz
python -u -m training.train_perceptron --epochs 200 --cache-data data/g400.npz --out models/e200.npz   # данные подгрузятся мгновенно
```

## Эксперименты (батч-замеры без GUI)

Директория `experiments/` — воспроизводимые замеры производительности и
силы игры без участия человека. Результаты (CSV + PNG-графики, 150 dpi)
складываются в `results/`.

```bash
python -m experiments.perf_benchmark   # Эксперимент A: скорость
python -m experiments.tournament       # Эксперимент B: сила игры
python -m experiments.analyze          # графики и сводные таблицы
```

- **Эксперимент A** (`perf_benchmark.py`) — время хода и счётчик
  (узлы / симуляции / прогоны) на 3 фиксированных позициях для разных
  параметров каждого алгоритма, по 5 повторов. Долгие конфиги (minimax
  depth=4, alpha-beta depth=5–6) ограничены таймаутом 60 c и пишутся как
  `NaN`. → `results/perf.csv`, `perf_summary.csv`.
- **Эксперимент B** (`tournament.py`) — круговой турнир: 6 пар × 30
  партий (15 за X, 15 за O), параметры по умолчанию (minimax/AB depth=3,
  MCTS 1000 симуляций, перцептрон — как есть). → `results/tournament.csv`.
- **Анализ** (`analyze.py`) — графики `perf_time_*.png`,
  `perf_branching.png` (эффективный коэффициент ветвления),
  `winrate_heatmap.png`, `avg_time_bar.png` и сводные таблицы в консоль.

### Как отслеживать и не потерять прогресс

Турнир может идти **десятки минут** (узкое место — медленный minimax
depth=3 в первой паре). Он устроен **отказоустойчиво**: каждая партия
сразу дописывается в `tournament.csv`, а повторный запуск **продолжает
с места обрыва** (уже сыгранные `game_id` пропускаются).

```bash
# следить за прогрессом (цель — 181 строка = 180 партий + заголовок)
wc -l results/tournament.csv

# чтобы macOS не уснул и долгий прогон не прервался — caffeinate:
caffeinate -is python -m experiments.tournament
```

> Не запускайте две копии одного эксперимента одновременно — оба пишут в
> один CSV. Если прогон прервался, просто запустите ту же команду снова —
> турнир до-играет недостающие партии.

## Структура проекта

```
game/        бесконечная доска (board.py) и правила победы (rules.py)
agents/      base.py + minimax / alphabeta / mcts / perceptron + heuristic
training/    генерация данных (data_generator.py) и обучение (train_perceptron.py)
metrics/     профайлер (profiler.py) и агрегатор статистики (stats.py)
gui/         pygame: board_view.py (доска+камера), sidebar.py (панель), main_window.py (цикл)
experiments/ батч-замеры: runner.py, perf_benchmark.py, tournament.py, analyze.py
tests/       pytest: доска, правила, эвристика, агенты, MCTS, метрики, перцептрон
models/      сохранённые веса перцептрона (.npz, в .gitignore)
data/        кэши датасетов (.npz, в .gitignore)
results/     CSV и PNG из experiments/
main.py      точка входа в GUI
```

## Тесты

```bash
python -m pytest
```

Покрывают доску, проверку победы, эвристику, базовое поведение агентов
(Alpha-Beta обыгрывает Random, MCTS обыгрывает Random и поисковый AB
малой глубины, перцептрон обыгрывает Random после обучения), метрики и
сам класс перцептрона.
